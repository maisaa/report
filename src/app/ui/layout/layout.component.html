Course Description:

This module is designed to develop student’s knowledge, understanding, and skills to solve problems through the creation of software solutions. It covers concepts and methods to develop the architectural design of medium-size software systems with sufficient complexity. It introduces fundamental design concepts and notations, and discusses different design methods with emphasis on Object Oriented design methods and techniques. The module also covers the construction of maintainable software systems using on of the dominant Object Oriented programming languages. During the course, students will undertake a term project working individually and in groups addressing the design and construction of a relatively complex software system.

Course Aims:

This course objective is to provide an overview of software analysis and design. The primary objective is to describe the meaning of software design and architecture, describe design programming paradigms, apply appropriate system design approaches that are either object-oriented or structural design, explain Solid design principles, design models using Solid design principles, describe and comapre traditional software development process and Model-Driven Architecture (MDA) development process, explain and apply MDA concepts, describe Object Constraint Language (OCL), explain OCL importance in software design, design models with OCL such as class diagram, interaction diagram, state charts diagrams and activity diagrams and explain the use of OCL for MDA.

Intended Learning Outcome:

Following the successful completion of this course, the student should be able to:

  1. Knowledge and Understanding:
    1. Define the meaning of software design and architecture.
  2. Subject Specific Skills
    1. Apply SOLID design principles.
    2. Apply design concepts of MDA.
  3. Cognitive and Intellectual Skill
    1. Diagram structured models
    2. Diagram object-oriented models
